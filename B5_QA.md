# B5 — 课堂压测清单

> 自动化冒烟通过后，再用 2～4 台设备跑一遍完整对局。

## 1. 课前环境

| 检查 | 命令 / 操作 | 期望 |
|------|-------------|------|
| 服务运行 | `pm2 status` 或 `npm start` | currency-safe online |
| Tunnel | `cloudflared tunnel run` | 域名可访问 |
| 健康 | `curl https://域名/health` | `{"ok":true,"backend":"ws"}` |
| 冒烟 | `cd server && node scripts/b5-smoke-test.mjs https://域名` | 全部 PASS |

## 2. 大厅流程

1. 设备 A（Host）创建房间，记下房间码  
2. Host 可选开启进房密码，设备 B 用错误密码应被拒绝  
3. 设备 B/C 以玩家加入不同队；设备 D 观战  
4. Host：**Shuffle 分队** → 地图图钉更新  
5. Host：**随机重分州属** → 州变、队员不变  
6. Host 开始比赛（或强制开始试讲）

## 3. 对局经济

1. 玩家 1、玩家 2 **同时**选同一外队州 → 部署  
2. 两人任务 Tab 应看到 **实拿预估**（份额随人数变化）  
3. 玩家 1 先完成三关并转账 → 队库增加、Activity 有记录  
4. 玩家 2 继续或重新部署，份额应按 **剩余奖池** 重算  
5. 等央行闪灯 → 点海域图钉 → 完成阅读 Bonus → 队库入账  

### 失误规则快查

| 关卡 | 计 1 次攻坚失误 |
|------|----------------|
| typing | 每错一字 |
| dial | **不计** |
| grid | 踩雷等（按现有关卡逻辑） |
| mastermind | **用尽行数刷新**（猜错只耗行） |
| fragsort | **排错并锁定**算 1 次 |

## 4. 断网重连

1. 对局中关闭设备 Wi‑Fi 约 10 秒  
2. 恢复网络后，界面应显示 **重连中** → **已连接**  
3. 无需刷新仍可在房；若状态异常，刷新后应回同一房  

## 5. 赛后

1. Host 结束比赛  
2. 检查 `server/data/history/` 有 `房间码_时间戳.json`  
3. `GET /api/history` 含本场摘要  
4. 战绩 Tab：**分队榜** + **个人贡献榜**  
5. Host 导出 CSV / 报告含个人贡献段
